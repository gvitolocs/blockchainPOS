Validation and measurements – answers

1. Verify signature generation and verification
   We can generate and verify a signature on a message. Test: sign a message, verify with same message → passes. Modify the message and verify again → verification fails. See rsa/signature_test.go TestVerifySignature. Run: cd rsa && go test -run TestVerifySignature -v

2. Hashing speed (bits per second)
   Message size: 10 KB = 10,000 bytes = 80,000 bits. Measured: 4,200 ns per 10 KB hash.
   Throughput: 80,000 / (4,200 × 10^-9) ≈ 1.95 × 10^10 bits/s ≈ 19 Gbit/s.
   Run: cd rsa && go test -bench=BenchmarkHashSpeed -benchmem

3. RSA signature generation time (2000-bit key)
   Signing a hash value (256 bits) with 2000-bit RSA key: 1,970,926 ns ≈ 1.97 ms per signature.
   Run: cd rsa && go test -bench=BenchmarkSignatureOnHash -benchmem

4. RSA throughput vs hashing
   If RSA were used directly on the whole message: at most 2000 bits per RSA operation. With ~1.97 ms per op, throughput ≈ 2000 / 0.00197 ≈ 1.14 × 10^6 bits/s ≈ 1.14 Mbit/s.
   Hashing: ~19 Gbit/s (from item 2). Hashing is far faster. Hash-then-sign: we sign the 256-bit hash once per message; hashing cost is negligible compared to RSA. Hashing greatly improves efficiency.

Commands (README style)

Run all RSA tests (key size, encrypt/decrypt, signatures, AES, hybrid):
  cd rsa
  go test

Run signature verification only:
  cd rsa
  go test -run TestVerifySignature -v

Run hashing speed benchmark (10 KB, ~4,200 ns → ~19 Gbit/s):
  cd rsa
  go test -bench=BenchmarkHashSpeed -benchmem

Run RSA signature benchmark (2000-bit key, ~1.97 ms per signature):
  cd rsa
  go test -bench=BenchmarkSignatureOnHash -benchmem

Run peer (P2P flooding + ledger) tests:
  cd peer
  go test

Run the peer demo (5 peers, 5 transactions each, ledger convergence check):
  cd peer
  go run .
